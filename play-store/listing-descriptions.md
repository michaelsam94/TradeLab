## Short description
Practice crypto paper trading with live charts and virtual orders.

## Full description
TradeLab is an Android crypto paper-trading simulator for learning technical analysis and practicing virtual order flows without using real money or sending real trades to an exchange.

What TradeLab does:

- Live markets: follow Binance public ticker data, search pairs, filter quote assets, and track favorites.
- Candlestick charts: review price action with timeframes, chart interactions, and technical overlays.
- Virtual trading: place simulated market and limit orders with a local virtual wallet, leverage simulation, take-profit, and stop-loss levels.
- Position monitor: keep simulated positions watched in the background with an ongoing notification and Stop action.
- Alerts: create factual price and indicator alerts with notification deep links back into the app.
- Indicator feed: review RSI, MACD, moving average, and momentum readouts with methodology notes.
- History and analytics: inspect filled virtual orders, realized P/L, win rate, max drawdown, and equity curve.
- Learn hub: read educational content that reinforces simulator-first usage.

How it works:

1. Open TradeLab and accept the educational simulator disclaimer.
2. Browse markets or open the Trade screen for a pair.
3. Place virtual orders against live public market prices.
4. Review simulated positions, alerts, indicators, and portfolio history.

Important simulator notice:

TradeLab is for educational and entertainment purposes only. It does not provide financial advice, investment advice, trading instructions, or real exchange execution. Virtual balances have no monetary value and cannot be withdrawn, exchanged, or converted to real currency or cryptocurrency.

Privacy:

TradeLab stores virtual portfolio, alert, settings, and market-cache data locally on the device. It reads public market data from Binance endpoints. No account system, ads, or analytics integration is configured in this repository.

Permissions:

Internet access is used to load public market data. Network state helps background jobs run when a connection is available. Notifications are used for alerts, daily digest, simulated position-close events, and the foreground position monitor.
